Umm, since when can a first order *heading* include a second order
*heading*
(the h1 element includes only the heading, not the other elements the
heading describes)?  According to XHTML 1.0 pr. rec. + HTML 4.01 rec, it
should be

<h1>First Order Heading</h1>There's no point using XHTML if you're going to change the meaning of the
elements that drastically.  If you want to make the markup more
hierarchical, I'd do XML like this:

<sec1 n="1"><h1>First Order Heading</h1>Order Heading</h2></sec1>

or XHTML like this:

<div name="sec1"><h1>First Order Heading</h1>="sec1.1"><h2>Second Order Heading</h2></div>

(I believe that a div element can contain another div element.)
