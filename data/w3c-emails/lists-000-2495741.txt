HTML Tidy versions 7 July 1999 and 26 July 1999 both appear to contain
errors in the ParseTagNames routine which prevents the correct
installation of tag names for new-empty-tags, new-inline-tags and
new-blocklevel-tags specified in a "config" file.  The tag names within
the tags.c module are stored in lower case but the ParseTagNames routine
converts new tag names to uppercase.  Also, the do...while loop should
test for (c != '\r' && c != '\n' && c != EOF ) instead of just ( c !=
EOF ) to terminate parsing of tag names and proceed to the next
property.

Gary Deschaines
