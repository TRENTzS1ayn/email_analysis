For all of these, we would like to have included:

- Whether or not it can be used as a MIME text/* character set
   (basically, whether CRLF is a 2-byte sequence encoded as 0x0D0A)
   (this is a requirement on the registration)

- If possible, pointers to a stable specification of the character sets,
   if possible with enough information to permit mapping tables to/from
   ISO 10646 to be constructed.
   (this is a strong recommendation)
