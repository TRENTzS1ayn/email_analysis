The current draft (may 2003) , states in [2.4.3 SequenceType] that
the default namespace for type names is the default element namespace.

It would be nicer for users to have a separate "default type
namespace", in the same way as there are a "Default element namespace"
and a "Default function namespace".

Initially, this default type namespace would be of course
"http://www.w3.org/2001/XMLSchema".

This way, the tedious xs:int, xs:string, xs:date etc. could be replaced
by int, string and date. Users of schemas would be free to redefine
the default type namespace the way they prefer.
