> I would like to suggest that the function specified in 10.2.5 be named
> get-in-scope-namespace-prefixes rather than as currently proposed.

We have decided to rename the function fn:get-in-scope-namespaces to
fn:get-in-scope-prefixes.

> Would there be any merit in adding a get-in-scope-namespace-uris()
> function?

We have decided to not add this function.  A user can use the function
fn:get-namespace-uri-for-prefix to obtain this information once they
have obtained the prefixes.

Please let us know if you are not satisfied with the resolution of this
comment.
