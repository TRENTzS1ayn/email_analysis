Chapter 20 of the XSLT 2.0 draft defines rules for using a default output
method in case the method attribute isn't specified. These rules
essentially detect HTML or XHTML.

I wonder why there isn't a similar rule for text: if the output doesn't
contain a single root element but rather starts with plain text then the
text output method could be chosen.
