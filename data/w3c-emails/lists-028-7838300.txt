> When I try to validate code as xhtml1.0 strict and within the img tag i
> have a border attribute. The code does not validate.
> 
> I was looking at the source of the www.w3.org home page and I noticed that
> the img tags for the xhtml1.0 strict logos at the bottom all include the
> border attribute. Yet the page still validates. Whats going on here?

The W3C home page is not XHTML 1.0 strict:

  <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
        "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

The logo at the bottom of the page indicates that the page is 
valid XHTML 1.0, but not that it is strict XHTML 1.0.
