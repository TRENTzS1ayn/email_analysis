Mimasa Ishikawa reports:

 "This is not a bug of the spec, but a problem of IE.  IE incorrectly
  tries to "sniff" media type regardless of the information provided
  by the Content-Type HTTP response header, and it incorrectly tries
  to parse DTD as if it is an XML instance.  You know, DTD syntax
  is not the same as XML instances, that's why IE failed."

Thanks Mimasa!
