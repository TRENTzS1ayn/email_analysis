Timely given the recent messages of the proliferation of URI schemes
and DanC's tel: noodling is Protozilla, http://protozilla.mozdev.org/
And see http://protozilla.mozdev.org/examples.html

Basically it looks like this will enable easy development in Mozilla of
handlers for URI schemes.  Excerpt:

"Protozilla automatically registers all protocols when the Protozilla
home page is loaded. For example, if the Perl script foo.pl is present
in the protocol directory, then the protocol foo: is automatically
registered. When an URI using this protocol needs to be loaded,
Protozilla executes the script foo.pl,"
