With the help of some of the w3c system guys I've finally got content 
negotiation on the namespace working, each of the following will get you a 
different result

wget -s http://www.w3.org/2000/09/xmldsig
#the spec in HTML

wget -s --header='Accept: text/html;' http://www.w3.org/2000/09/xmldsig
#the spec in HTML

wget -s --header='Accept: text/xml;' http://www.w3.org/2000/09/xmldsig
#the schema

wget -s --header='Accept: application/xml-dtd;' 
http://www.w3.org/2000/09/xmldsig
#the DTD

I've also been considering RDDL [1], but it and the schema would vie for the 
same content type and isn't that widely used yet -- thought it is cool and 
would obviate the necessity of tricky URI rewrites!

[1] http://www.openhealth.org/RDDL/
