I see the following steps that the WG should aim to take regarding
datatypes:

1. Vote whether a single labeled node denotation of datatype
   values is to be adopted, in some form, replacing the
   datatype property idiom as defined by the latest datatyping
   WD

2. Agree whether there are sufficient technical reasons why
   that single labeled node denotation cannot be a URIref
   node

   a. If not, then agree on a URI encoding for such values
   b. If so, then agree on a representation for the new
      type of labeled node

3. Decide whether non-datatyped literals have tidy or untidy
   interpretation

4. Decide if and how datatyping range constraints will be 
   asserted in terms of the above decisions regarding 
   datatyped and non-datatyped literals
