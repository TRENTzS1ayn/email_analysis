Task  12. JA: Create dependency list for user agent and authoring tools

below are dependencies or possible dependencies in the Authoring Tools
accessibility Guidelines [http://www.w3.org/TR/1999/WAI-AUTOOLS-19990903]
that impact the User Agent Accessibility Guidelines
[http://www.w3.org/WAI/UA/WAI-USERAGENT-19990827/]

the AUAG Checkpoints with dependencies are each listed followed by
dependency prefaced with DEP:

Guideline 2. Generate standard markup

DEP: UA should render all standard markup

AUAG Checkpoints 2.1 Use applicable W3C Recommendations. [Priority 2]

DEP: UA should render all appropriate W3 recommended markup in an
accessible manner

AUAG Checkpoints 2.2 Ensure that markup is generated in accordance with a
published specification [Priority 1]

DEP: UA should render appropriate markup in accordance with a published
specification in an accessible manner

AUAG Checkpoints 2.3 Ensure the tool produces markup in a language that
enables accessibility [Priority 1]

DEP: UA should render language appropriate accessibility markup in an
accessible manner

AUAG Checkpoints 3.1 Prompt the author to provide alternative information
(e.g., captions, expanded versions of acronyms, long descriptions of
graphics). (Priority 1 for alternative information that is
[Web-Content-Priority-1], Priority 2 for alternative information that is
[Web-Content-Priority-2], Priority 3 for alternative information that is
[Web-Content-Priority-3])

DEP: UA should render (or a navigation method to) appropriate alternative
information in an accessible manner

AUAG Checkpoints 4.5 Allow the author to transform presentation markup that
is misused to convey structure into structural markup, and to transform
presentation markup that is stylistic into style sheet markup.. [Priority 3]

DEP: UA should render appropriate markup and stylesheets in an accessible
manner
