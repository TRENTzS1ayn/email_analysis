Daniel Dardailler wrote:
> 
> > JW:: Without re-reading the HTML specification on this point, I am under
> > the impression that any markup inside the PRE element (prior to the end
> > tag of PRE itself) is treated as part of the preformatted text rather than
> > as markup.
> 
> No it's OK, you can have CODE in PRE:
> 
> <!ENTITY % pre.exclusion "IMG|OBJECT|BIG|SMALL|SUB|SUP">
> 
> <!ELEMENT PRE - - (%inline;)* -(%pre.exclusion;) -- preformatted text -->
> <!ATTLIST PRE
>   %attrs;                              -- %coreattrs, %i18n, %events --


A better solution would be to separate the semantics
from the presentation by using the "white-space" property:

<STYLE type="text/css">
CODE.mylanguage { white-space: pre }
</STYLE>

<CODE class="mylanguage">
  BEGIN block
     for i in 1 to 10, do
     ...
     end for
  END block
</CODE>

However, this would only work for CSS-enabled browsers.
