John Slatin:
> 
> Maybe Yvette has hit upon another test for distinguishing layout 
> tables from data tables:
> 
> If the same effect could be achieved using CSS, it's a layout table.

That's the test I always use myself: Would I loose semantics by converting
this table to CSS? If so, it's a data table. If not, it's a layout table.

This is done with in the back of my mind "... if CSS had all the features I
wished it had, if CSS was as well supported by browsers as I wish it was,
and if I was as good at using CSS as I wish I was"
