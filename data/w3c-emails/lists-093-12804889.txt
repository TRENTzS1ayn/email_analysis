We have detected in Amaya 1.4 an annoying bug: if one creates a table when
the current selection points to an empty paragraph, Amaya tries to replace
the empty paragraph by the table, but there is a bug which corrupts Amaya
internal structure.
This bugs is now fixed, so we plan to make a new complete release (Windows and
Unix versions) beginning of January.
In current version ou can get round this bug by inserting a character into 
the empty paragraph before creating the table.
Sorry for the inconvenience.
