On Feb 12,  3:22pm, Glenn Randers-Pehrson ARL-WMRD-TED-TIB wrote:


> In a PNG, with
>
>      PLTE 0 0 0   0 0 0    255 255 0
>      tRNS 0
>
> Amaya treats both of the 0 0 0 colors as transparent.  Evidently it
> promotes indexed pixels and the tRNS chunk, and then makes everything
> with 0 0 0 pixels transparent.

Oops.

> That's incorrect.  Transparency is by index values, not by colors, in
> indexed-color PNGs.

Right. Presumably this is handled by the application and not by libpng,
because Amaya uses zlib and libpng. Has anyone looked over the source?

This may be related to Amaya 0.95b using an 8bit indexed internal
color model.

> Also, Amaya doesn't seem to understand the < object > tag.

Correct, the released version does not implement the object tag yet.
