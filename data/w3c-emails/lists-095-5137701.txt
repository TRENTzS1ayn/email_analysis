This bug is old but has not yet been fixed in Amaya 5.0 (Linux binary
distribution):

Wenn browsing a directory in the "File / Open Document" dialogue and the
directory contains a dead link (symbolic link that points to a nonexisting
file), only the files in the directory until before the dead link are shown.
Files after the dead link (in lexicographical order) are not shown.
