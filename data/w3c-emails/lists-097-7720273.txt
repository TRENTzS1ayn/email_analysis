Since the rdfxml-mediatype is now a "Informational RFC" [1] I'd propose
to change the annotea protocol to use application/rdf+xml instead of
application/xml (for accept header and return content-type).

The motivation is to allow server to return a xml response optimized for
being rendered in browsers (with a stylesheet PI) when the client (such
as mozilla) indicates in the accept-header that xml is the preffered
response-type, such a response does not have to be valid RDF/XML.
