The w3c-synd stylesheet [1] has a bug; it uses a variable reference in a 
template's match attribute, specifically,

>  <xsl:template match="h:html [h:head/@profile=$Profile]">

According to XSLT [2], this is an error; "It is an error for the value 
of the match attribute to contain a VariableReference." Saxon certainly 
doesn't like it.

I think the fix is to remove the Profile variable and hard-code it into 
the match, since this is the only place it's used in the stylesheet.
