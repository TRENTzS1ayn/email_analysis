Hi,

Some comments on the XForms 1.0 test suite,

  <http://www.w3.org/MarkUp/Forms/Test/>

There are two files in the test suite which name only differs in case,
in particular

  <http://www.w3.org/MarkUp/Forms/Test/chapter3/C3-002.xml>
  <http://www.w3.org/MarkUp/Forms/Test/chapter3/c3-002.xml>

These files are also in the .zip archive which makes the test
inaccessable on systems with case-insensitive file systems.

Most of the overview pages are not valid HTML, see e.g.

  <http://www.w3.org/MarkUp/Forms/Test/chapter3/,validate>
  <http://www.w3.org/MarkUp/Forms/Test/,validate>

A number of files link to member-only resources, see e.g.

  <http://www.w3.org/MarkUp/Forms/Test/chapter3.html,checklink>

The @namespace declaration in 

  <http://www.w3.org/MarkUp/Forms/Test/test.css>

file is invalid, @namespace must precede all rulesets, see 

  <http://www.w3.org/TR/css3-namespace/>

I have mixed feelings about the use of proprietary featurs in a test
suite style sheet...

regards.
