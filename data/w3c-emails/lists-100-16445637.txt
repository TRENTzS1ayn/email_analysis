I'm not sure where the group is with respect to categorizing tests.
Talking with Dimitris he asked me to send my ideas to this list.

I propose that we do something like this:
<test domLevel="2" domModule="Events" domFeature="Mouse"
appliesToMarkup="HTML" resultTesting="Exception" ... />

appliesToMarkup could be:
HTML Only
XML Only
Both

resultTesting could be:
Exception
ValueCheck
Etc...

This would allow test harnesses to filter tests for running and
categorize reporting of results.
