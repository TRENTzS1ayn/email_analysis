Not an error. The IDL name of this value is "attributes", but when
implemented in a programming language -- in this case, Java -- it's
implemented as a getAttributes() method. The same is true for the other IDL
attributes such as nodeName/getNodeName(), firstChild/getFirstChild(), and
so on.
